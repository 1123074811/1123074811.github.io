Access to video at ‘https://iblog-file.oss-cn-beijing.aliyuncs.com/xxx.mp4‘ from origin ‘http://localhost:5173‘ has been blocked by CORS policy: No ‘Access-Control-Allow-Origin’ header is present on the requested resource.
===========================

这说明：前端页面（http://localhost:5173）试图访问阿里云 OSS 上的视频资源时，由于 OSS 未配置允许该域名的跨域规则，浏览器出于安全限制阻止了访问，导致视频无法回显。

二、解决方法：配置 OSS 跨域规则

需要登录阿里云 OSS 控制台，为存储视频的 Bucket 配置跨域访问规则，允许前端域名（如http://localhost:5173）访问资源。

具体步骤：

  1. 登录阿里云 OSS 控制台进入 阿里云 OSS 管理控制台，找到你的 Bucket（如iblog-file）。

  2. 进入跨域设置在 Bucket 左侧菜单中，找到「权限管理」→「跨域资源共享（CORS）」，点击「创建规则」。

  3. 配置跨域规则按以下内容填写（根据你的实际情况调整）：

    1. 来源（Origin）：允许访问的前端域名，开发环境填 http://localhost:5173，生产环境填你的网站域名（如https://xxx.com），也可临时用 * 允许所有域名（不推荐生产环境）。

    2. 允许 Methods：至少勾选 GET（视频播放需要）、PUT、POST（上传可能需要）。

    3. 允许 Headers：填 *（允许所有请求头）。

    4. 暴露 Headers：填 ETag（可选，根据需要添加）。

    5. Max-Age（秒）：建议设为 3600（缓存 1 小时，减少重复验证）。

  4. 保存规则点击「确定」完成配置，生效时间约 1-2 分钟。

三、其他可能的辅助调整

  1. 检查视频 URL 是否正确确认前端form.video存储的是 OSS 返回的完整 URL（如https://bucketName.endpoint/文件名），可在控制台打印form.video的值验证。

  2. 富文本编辑器高度警告控制台提示「编辑区域高度 < 300px 可能导致定位异常」，建议调整编辑器高度：

  3. css

    /* 在样式中增加 /:deep(.w-e-text-container) {height: 400px !important; / 确保高度 ≥ 300px */}

  4. Vue 组件事件警告关于updateAccount事件的警告不影响视频回显，可忽略或在对应组件中通过emits声明自定义事件消除警告。

总结

视频回显失败的根本原因是 OSS 跨域配置缺失，优先配置 OSS 的 CORS 规则允许前端域名访问。配置完成后，浏览器将允许加载 OSS 上的视频资源，回显功能即可正常工作。
