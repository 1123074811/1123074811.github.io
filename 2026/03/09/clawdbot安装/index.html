windows环境准备

  1. 准备一台windows电脑，配置要求不高，比如我的是windows10，8G，I5处理器

  2. 安装node.js，需要22版本以上： https://nodejs.org/en 官网下载安装，具体这里就不赘述了，按着操作处理就好

clawdbot安装

安装clawdbot

powershell执行

iwr -useb https://molt.bot/install.ps1 | iex

这里可能出现第一个报错，这个错误是因为 PowerShell 的执行策略（Execution Policy）阻止了脚本运行。你需要修改执行策略来允许安装脚本运行：

直接在powershell 中执行 Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process

接下来我这又出现了报错，看上去是nup和git拉取代码的问题：

清除缓存，重试成功：
npm cache clean –force

安装过程配置

正常情况下，会出现配置界面，首先是风险提示，选yes（否则无法继续，直接结束~）：

Onboarding mode 选择 QuickStart，config handling 选Update values

接下来是关键的模型这部分，我这里选的是MiniMax，毕竟Clawdbot 项目作者 Peter Steinberger也推荐MiniMax,对国内用户也友好（原作者使用MiniMax，但千问有免费额度且注册方便，建议使用）：

接下来选具体的模型：我这里选的是M2.1:

选择完之后会让你输入API-key，这会就需要到minimax官网去订阅了，具体如下，我订的是29这个试试水，然后复制对应的apikey进来： https://platform.minimaxi.com/subscribe/coding-plan

把api-key贴上去：

接下来的步骤都可以按照默认或者跳过：

这个是选择频道，可以先不处理，有网页端界面，后续可以用飞书等等：

skill相关的也可以先看看：

其他的全部NO就行：

至此，就配置好了，可以启动试试了~
clawdbot启动测试

启动命令
clawdbot gateway –port 18789

启动后输入内容无回复：

这是 API 请求地址没有配置对，需要和上面订阅的 CodingPlan 保持一致，直接在 Powershell 命令行输入以下命令，然后重启就OK了：
clawdbot config set models.providers.minimax.baseUrl “https://api.minimaxi.com/anthropic“

成功对话

下面就可以成功对话了，接下来还会去测试一些有意思的功能和接入其他频道测试~

# 重新配置clawdbot向导clawdbot onboard# 启动clawdbotclawdbot gateway –port 18789# 终止和重启clawdbot gateway stopclawdbot gateway restart
