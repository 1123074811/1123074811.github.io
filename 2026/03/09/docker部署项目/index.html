*安装Docker（yum安装）*

基于 CentOS 7 安装

  1. 官网安装参考手册：

CentOS | Docker Docs

  1. 确定你是CentOS7及以上版本

    [root@192 Desktop]# cat /etc/redhat-releaseCentOS Linux release 7.2.1511 (Core)

  2. 备份原有源配置：

    mv /etc/yum.repos.d/CentOS-Base.repo /etc/yum.repos.d/CentOS-Base.repo.bak

  3. 国内阿里源归档：

    curl -o /etc/yum.repos.d/CentOS-Base.repo https://mirrors.aliyun.com/repo/Centos-7.repo

  4. yum安装gcc相关（需要确保当前主机可以上网 ）

    yum -y install gccyum -y install gcc-c++

  5. 卸载旧版本（非必须，如果没有安装过的话，这一步骤忽略）

    yum remove docker \ docker-client \ docker-client-latest \ docker-common \ docker-latest \ docker-latest-logrotate \ docker-logrotate \ docker-engine

  6. 安装需要的软件包

    yum install -y yum-utils

  7. 设置stable镜像仓库

正确推荐使用阿里云的
yum-config-manager –add-repo http://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo

  1. 更新yum软件包索引

    yum makecache fast

  2. 安装Docker CE

    yum install docker-ce docker-ce-cli containerd.io docker-compose-plugin

如果安装失败则卸载已下载的不完整包：
yum remove -y docker-ce docker-ce-cli containerd.io

安装依赖工具：
yum install -y yum-utils

添加 Docker 官方源（适配 CentOS 7）：
yum-config-manager –add-repo https://download.docker.com/linux/centos/docker-ce.repo

安装指定版本的 Docker（避免最新版适配问题）：
yum install -y docker-ce-20.10.24 docker-ce-cli-20.10.24 containerd.io docker-compose-plugin

  1. 启动docker

    systemctl start docker

  2. 由于直接从 Docker Hub 拉取镜像在国内常常速度很慢，推荐配置多个国内镜像源作为 registry-mirrors，挂到 Docker 的 daemon 配置中（记得收藏也许后面能用得着）。

编辑或创建 /etc/docker/daemon.json，例如：
{ “registry-mirrors”: [ “https://docker.registry.cyou“, “https://mirror.aliyuncs.com“, “https://docker.m.daocloud.io“, “https://docker.mirrors.ustc.edu.cn“, “https://mirror.baidubce.com“ ], “max-concurrent-downloads”: 10, “max-concurrent-uploads”: 5}

保存后重启 Docker：
systemctl daemon-reloadsystemctl restart docker

你可以通过 docker info 查看是否生效：
docker info | grep -i “Registry Mirrors” -A 5

  1. 测试

    docker versiondocker run hello-world

查看镜像
docker images

显示出镜像，到此就算docker搭建成功了
docker常用命令

  1. 拉取镜像docker pull，注意拉取的镜像先到镜像源中查看是否存在，但是DockerHub非常难访问，国内有加速源如毫秒镜像：https://1ms.run/

例如：
docker pull mysql:latest

  1. 查看镜像docker images

  2. 删除镜像docker rmi（加 -f 强制删除）,例如：

    docker rmi my-app

  3. 构建镜像docker build，例如：

    docker build -t my-app .

my-app是镜像名

. 是在当前目录下搜索是否有Dockerfile文件，如果有就构建镜像

  1. Dockerfile文件编写（后端基于jdk镜像，前端基于nginx镜像）

    FROM library/openjdk:21-ea-21-jdk-oracle

    设置工作目录

    WORKDIR /app

    暴露端口（与应用配置保持一致）

    EXPOSE 9090

    复制 Maven 打包后的 JAR 到容器中并命名为 app.jar

    注意：这里的路径是容器构建上下文中的路径，一般为 target 目录下的最终 JAR

    COPY springboot-0.0.1-SNAPSHOT.jar app.jar

    启动命令

    ENTRYPOINT [“java”, “-jar”, “app.jar”]

  2. 启动容器（或者说创建容器）：docker run，例如：

    docker run -d -p 3306:3306 –name mysql -v /mydata:/var/lib/mysql -e MYSQL_ROOT_PASSWORD=ojc132598. mysql:latest

  3. 查看容器

    docker ps

  4. 启动 / 停止 / 重启容器docker start/stop/restart跟容器名

  5. 进入运行中的容器查看并修改配置1. 列出容器： docker ps （找到nginx容器的NAME或CONTAINER ID）2. 进入容器终端： docker exec -it <容器名或ID> /bin/bash  3. 查看主配置： cat /etc/nginx/nginx.conf  4. 查看子配置：ls /etc/nginx/conf.d/  或  cat /etc/nginx/conf.d/default.conf  5. 退出容器： exit

    编辑源文件(添加国内镜像源)echo “deb http://mirrors.aliyun.com/debian/ trixie main non-free non-free-firmware contribdeb-src http://mirrors.aliyun.com/debian/ trixie main non-free non-free-firmware contribdeb http://mirrors.aliyun.com/debian-security/ trixie-security maindeb-src http://mirrors.aliyun.com/debian-security/ trixie-security maindeb http://mirrors.aliyun.com/debian/ trixie-updates main non-free non-free-firmware contribdeb-src http://mirrors.aliyun.com/debian/ trixie-updates main non-free non-free-firmware contrib” > /etc/apt/sources.list# 重新更新源并安装Vimapt update && apt install -y vimvim /etc/nginx/nginx.conf

    server { listen 80; server_name localhost; root /usr/share/nginx/html; index index.html; location / { try_files $uri $uri/ /index.html; }}

修改后退出输入 docker exec vue nginx -s reload

如果需配置域名参考：

  1. 删除容器

    停止容器（替换为容器名或容器ID，支持部分ID，如前3位）docker stop <容器名/容器ID># 删除容器docker rm <容器名/容器ID>

  2. 查看容器日志

    docker logs -f <容器名/容器ID>

上传项目

  1. 先将项目中的IP地址修改成服务器地址，后端项目maven中要添加打包插件：

    org.springframework.boot spring-boot-maven-plugin 3.x.x repackage

这里是需要的

  1. 前端上传dist目录和Dockerfile文件，后端上传jar包和Dockerfile文件，运行数据库容器后通过远程连接工具导入数据库文件

  2. 解压文件：

如果是linux文件直接用命令
tar -xvf 文件名.tar

如果是zip文件，先安装依赖
sudo yum install unzip

再执行解压命令
unzip 文件名.zip

  1. 记得开启阿里云服务器安全组访问端口，如后端9090，有Oss的开启跨域配置

  2. 运行完前后端项目以及数据库容器就部署成功了！
